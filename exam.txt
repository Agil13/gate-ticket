Написать клиент-серверное приложение для генерации пропусков через турникеты.
Общий взгляд на систему - состоит из 2х модулей:
1. Веб приложение со следующей функциональностью:
	- Вход в систему с установленными в config файле логином и паролем
	- CRUD-панель для управления турникетами
	- CRUD-панель для управления посетителями
	- API-обработчик, который принимает ID турникета и посетителя и решает, есть ли у указанного посетителя право проходить через указанный турникет
	- Генерация HTML или PDF пропуска для выбранного посетителя
	
2. Турникет, который оснащен сканером QR-кодов:
	- при считывании QR-кода с пропуска отправляет запрос на сервер, и если сервер дал добро, пропускает посетителя.
	

Вам нужно разработать первый модуль - веб приложение и сервер для него.

Пример использования системы:
1. Оператор заходит на страницу логина.
2. Оператор вводит логин и пароль и нажимает кнопку "войти".
3. Сервер проверяет правильность введенных данных. Если данные правильны - редиректит пользователя на страницу посетителей. Если нет - оставляет на странице логина.
4. Оператор попадает на страницу посетителей.
5. Оператор добавляет нового посетителя.
6. Оператор переходит на страницу турникетов.
7. Оператор добавляет новый турникет.
8. Оператор попадает на страницу посетителей.
9. Оператор редактирует посетителя, добавляя ему доступ к турникету.
10. Оператор нажимает на посетителе кнопку "сгенерировать пропуск".
11. Открывается страница с пропуском, который Оператор может распечатать или переслать посетителю.
12. Оператор нажимает кнопку "выход" и выходит из системы. Попадает на страницу логина.



Модели:

Gate:
	ID: string
	Name: string
	Comment: string
	
Visitor:
	ID: string
	Name: string
	Photo: string
	Gates: number[]
	
API:
	GET /login - получить страницу логина
	POST /login - отправить логин и пароль для проверки
	
	POST /logout - выход
	
	GET /visitors - получить страницу посетителей
	POST /visitors - создать посетителя
	PUT /visitors/{id} - отредактировать посетителя с указанным id
	DELETE /visitors/{id} - удалить посетителя с указанным id
	
	GET /gates - получить страницу турникетов
	POST /gates - создать турникет
	PUT /gates/{id} - отредактировать турникет с указанным id
	DELETE /gates/{id} - удалить турникет с указанным id
	
	POST /check - проверить, есть ли у посетителя доступ к турникету

	GET /pass/{id} - сгенерировать пропуск для посетителя с указанным id


Пропуск должен содержать всю информацию о посетителе. ID пользователя должен быть отображен с помощью QR-кода.
